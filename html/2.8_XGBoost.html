0. Introduccin

En este documento, nos centraremos en la creacin de un Modelo XGBoost.

XGBoost es la abreviatura de las palabras inglesas "extreme gradient boosting" (refuerzo de gradientes extremo). Este mtodo se basa en rboles de decisin y supone una mejora sobre otros mtodos. Funciona bien con datasets grandes y complejos al utilizar varios mtodos de optimizacin.

Una vez creado el modelo, procederemos a optimizarlo con los valores de train y predecir los valores de validacin.

3. Creacin del modelo

Para la creacin del modelo XGBOOST utilizaremos el algoritmo XGBClassifier de la libreria XGBoost.

Creamos un Pipeline que junte el preprocesado y el algoritmo _XGBClassifier_para ejecutarlo en el orden mencionado.

Para el algoritmo designaremos los parmetros por default, pero utilizaremos todo el procesador para agilizar el proceso. En el caso de que obtenga muy buen resultado, estudiaresmo como modificar los hiperparmetros para tratar de mejorarlo.

In [5]:
# Design the parameters.

n_jobs = -1
random_state = 12345

# Create a pipeline that applies a preprocess of the data and then the LogisticRegression model.

xgb_model = Pipeline([
    ('preprocessor', preprocessor),
    ('clasificador', XGBClassifier(n_jobs=n_jobs, random_state=random_state)
    )
])

Como ya comentamos en el documento 02_DummyModel, nuestro dataset se caracteriza por tener un desbalanceo muy grande, es por ello que el anlisis de las mtricas de accuracy y precisin siempre van a ser muy elevadas, sin embargo, no reflejan la realidad predictora de nuestro modelo. Por lo que nos centraremos principalmente en las mtricas F2-score y AUC .

  • El modelo nos calcula un Recall de 0.91, valor alto, lo que significa que el modelo llega a detectar correctamente aquellas transacciones que son fraude o no.

  • El modelo nos calcula un F2 score de 0.84, el valor ms alto de todos los modelos que hemos estudiado, lo cual es una gran opcin a ser escogido para el proceso de prediccin sobre el set de test.

6.3. Curva ROC¶

Para evaluar nuestro modelo, la representaremos graficamente para deducir que tan bueno es el modelo.

In [14]:
# Keep probabilities for the positive outcome only
yhat = predict_probabilities[:, 1]
# Calculate roc curves
fpr, tpr, thresholds = roc_curve(y_val, yhat)
# Plot the roc curve for the model
plt.plot([0,1], [0,1], linestyle='--', label='No Skill')
plt.plot(fpr, tpr, marker='.', label='Gradient Boosting')
# Axis labels
plt.xlabel('False Positive Rate')
plt.ylabel('True Positive Rate')
plt.legend()
# Show the plot
plt.show()

# Display the ROC AUC.

print('The Compute Area Under the Receiver Operating Characteristic is '+ str(metrics.roc_auc_score(y_val,yhat)))

The Compute Area Under the Receiver Operating Characteristic is 0.9977765228566379

Adems de haber obtenido un gran score en la mtrica F2, el AUC es extremadamente bueno, por lo que es un gran candidato para predecir el set de test..
