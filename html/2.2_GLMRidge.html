0. Introduccin

En este documento, nos centraremos en la creacin de un Modelo Lineal generalizado (GLM).

Este modelo se emplea para predecir valores de una variable continua (Y) como una combinacin lineal de las variables predictoras (X). Un modelo GLM est compuesto por tres partes:

  • Una componente aleatoria. Identifica la variable respuesta y su distribucin de probabilidad.

  • Una componente sistemtica. Especifica las variables explicativas (independientes o predictoras) utilizadas en la funcin predictora lineal.

  • Una funcin link. Es una funcin del valor esperado de Y , E(Y ), como una combinacin lineal de las variables predictoras

Una vez creado el modelo, procederemos a optimizarlo con los valores de train y predecir los valores de validacin, para posteriormente anlizar sus mtricas.

3. Creacin del modelo

Para la creacin del modelo, utilizaremos un modelo de regresin logstica dado que es un GLM donde la distribucin de probabilidad puede ser Bernoulli o Binomial. Por lo tanto la estimacin de los parmetros y los contrastes de hiptesis utilizan la teora desarrollada para los GLM's.

Creamos un Pipeline que junte el preprocesado y el modelo de regresin para ejecutarlo en el orden mencionado.

Designaremos una penalizacin de tipo L2 (Ridge), dado que en el documento 04_LogisticRegressionLasso realizaremoss una regresin con un modelo L1 (Lasso).

In [5]:
# Designate the hyperparameters.

penalty = 'l2'
n_jobs = -1
random_state = 12345

# Create a pipeline that applies a preprocess of the data and then the LogisticRegression model.

glm_model = Pipeline([
    ('preprocessor', preprocessor),
    ('clasificador',  LogisticRegression(penalty=penalty,n_jobs=n_jobs, random_state=random_state)
    )
])

Como ya comentamos en el documento 02_DummyModel, nuestro dataset se caracteriza por tener un desbalanceo muy grande, es por ello que el anlisis de las mtricas de accuracy y precisin siempre van a ser muy elevadas, sin embargo, no reflejan la realidad predictora de nuestro modelo. Por lo que nos centraremos principalmente en las mtricas F2-score y AUC .

  • El modelo nos calcula un Recall de 0.65, dado que el modelo no llega a detectar correctamente aquellas transacciones que son fraude o no.

  • El modelo nos calcula un F2 score de 0.35, un valor mayor que en el modelo Dummy, sin embargo, sigue siendo un valor muy bajo como para realizar predicciones ptimas.

Para evaluar nuestro modelo, la representaremos graficamente para deducir que tan bueno es el modelo.

# Keep probabilities for the positive outcome only
yhat = predict_probabilities[:, 1]

# Axis labels
plt.xlabel('False Positive Rate')
plt.ylabel('True Positive Rate')
plt.legend()
# Show the plot
plt.show()

# Display the ROC AUC.

print('The Compute Area Under the Receiver Operating Characteristic is '+ str(roc_auc_score(y_val,yhat)))

The Compute Area Under the Receiver Operating Characteristic is 0.9558578554515484

Una vez representada la curva ROC, la cual tiene un rea bajo la curva con valor de 0.95, observamos como nuestro modelo aparentemente tiene un alto nivel de discriminacin, sin emabargo, nos interesa penalizar aquellas predicciones que han sido fraudulentas y las ha considerado correctas.

Por lo que a partir de este momento, consideraremos la mtrica F2 como la principal para basarnos si nuestros modelos son certeros o no.
